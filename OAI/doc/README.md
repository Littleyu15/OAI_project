## OAI 5GC-gNB-nrUE Architecture

當一個 5G 設備要求上網時，`AMF` 處理連線，`AUSF`/`UDM`/`UDR`/`MySQL` 確認身分，`SMF` 建立上網通道並發配 IP，最後由 `UPF` 負責將設備的資料轉到 `Ext-DN` 完成上網。

協定縮寫|英文全名與中文|核心角色|主要功能說明
|---|---|---|---|
NAS|Non-Access Stratum (非接取層)|與核心網溝通的大腦|處理門號認證、IP 取得與移動性管理。基地台不參與此層的解讀。
RRC|Radio Resource Control (無線資源控制)|基地台連線的總指揮|負責手機與單一基地台的連線建立、訊號測量與換手 (Handover) 控制。
PDCP|Packet Data Convergence Protocol (封包數據聚合協定)|封包加密與壓縮|負責資料加密、防篡改，並壓縮 IP 標頭以節省珍貴的空中頻寬。
RLC|Radio Link Control (無線鏈路控制)|壓縮/加密後封包切割與重組|依據底層傳輸能力將大封包切割或合併，並負責遺失片段的重傳。
MAC|Medium Access Control (媒介接取控制)|資源排程的調度|決定多台設備之間傳送資料的時機與優先順序，動態分配頻譜資源。
PHY|Physical Layer (實體層)|訊號傳送|將數位資料轉換為無線電波 (調變與編碼) 透過天線發射，並進行硬體除錯。
